## Component Inventory

### Floating Icon (Phase 1)

**Component:** `BagIcon.tsx`

**Visual specification:**
- Size: 48x48px (fixed)
- Shape: Circle with subtle border
- Background: #6B7280 (gray-500) with 20% opacity
- Border: 1px solid #9CA3AF (gray-400)
- Shadow: `0 4px 12px rgba(0, 0, 0, 0.15)`
- Position: `fixed` with z-index: 2147483647
- Default position: Bottom-right corner (20px from right, 20px from bottom)

**States:**
1. **Idle (default):** Gray-500 background, normal shadow
2. **Hover:** Gray-400 background, cursor: grab
3. **Dragging:** Blue-500 background (#3B82F6), cursor: grabbing, enhanced shadow `0 8px 24px rgba(59, 130, 246, 0.3)`
4. **Snapping:** Brief scale animation (1.0 → 1.1 → 1.0) when snapping to edge

**Interaction behavior:**
- Drag to any of 4 edges (top/bottom/left/right)
- Snap to edge when released within 50px threshold
- Position persists to chrome.storage.local
- No click interaction in Phase 1 (clicking does nothing until Phase 2)

**Technical constraints:**
- Must be direct child of document.body (CONTEXT.md D-11)
- Must use Shadow DOM isolation (CONTEXT.md D-04)
- Must use position: fixed for viewport-relative positioning

## Implementation Notes

1. **Shadow DOM setup:** Use WXT's `createShadowRootUi` with `cssInjectionMode: 'ui'` (RESEARCH.md Pattern 1)
2. **Drag implementation:** Use react-draggable 4.5.0 with custom edge-snapping logic (RESEARCH.md Pattern 3)
3. **Position storage:** Use WXT storage module with `IconPosition` interface (RESEARCH.md Pattern 2)
4. **Z-index:** Always use 2147483647 on icon container (CONTEXT.md D-10)
5. **Boundary constraints:** Clamp position to viewport bounds to prevent off-screen icon (RESEARCH.md Pitfall 4)
