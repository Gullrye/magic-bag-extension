## Component Inventory

### Floating Icon (国风 redesign)

**Component:** `BagIcon.tsx`

**Visual specification:**
- Size: 48x48px (unchanged from Phase 1)
- Shape: Traditional Chinese pouch/bag silhouette with gathered top
- Primary body: red-700 with subtle fabric texture gradient
- Gold trim at the gathered top (pouch opening)
- Optional: small tassel element at bottom in gold
- Optional: subtle jade-colored seal/stamp decoration
- Drop shadow: ink-colored (rgba(30,30,30,0.4)) for visibility on all backgrounds

**States:**
1. Idle: pouch shape with subtle shadow, gold trim visible
2. Hover: slight scale increase (1.05x), enhanced shadow, gold elements brighten
3. Dragging: scale (1.1x), enhanced shadow, cursor changes to grabbing
4. Active (grid open): subtle glow or highlight indicating expanded state

**Interaction behavior:**
- Click (without drag): toggle grid open/close
- Drag: reposition icon, snap to nearest screen edge on release
- Position persists across sessions

**Implementation approach:**
- Use inline SVG for the pouch shape to allow precise styling
- CSS transitions for hover and drag states
- Maintain existing drag behavior from react-draggable

### Options Page

**Component:** `entrypoints/options/index.tsx` (new)

**Visual specification:**
- Full-page layout with amber-50 background matching bag panel
- Centered content container max-width 480px
- Page title at top: "法宝袋设置" (24px, semibold)
- Two sections stacked vertically with 24px gap:

  2. Import section
- Each section: white surface with subtle border, rounded corners (12px), padding 24px
- Section title: 16px semibold, amber-900

**Layout structure:**

## Interaction Patterns

### Export Flow
1. User opens extension options (right-click icon -> Options, or chrome://extensions)
2. User sees "法宝袋设置" page with export section
3. User clicks "导出 JSON" button
4. JSON file downloads to user's default download location
5. Success toast appears showing count of exported tabs

### Import Flow
1. User opens extension options page
2. User clicks "选择文件" button in import section
3. System file picker opens (filtered to .json files)
4. User selects a valid JSON file
5. Tabs are merged into existing storage (duplicates skipped silently)
6. Success toast appears showing count of imported tabs

### Icon Redesign
1. User sees new 国风 styled pouch icon on all pages
2. Hover reveals subtle animation (scale, shadow enhancement)
3. Drag behavior unchanged from Phase 1
4. Click behavior unchanged from Phase 2

## Implementation Notes

1. **Options page location:** Create `entrypoints/options/index.tsx` following WXT convention for extension options pages.
2. **Icon implementation:** Replace the placeholder circle in `BagIcon.tsx` with an inline SVG pouch shape. Maintain all existing drag/position logic.
3. **Export format:** Array of SavedTab objects as specified in CONTEXT.md D-01.
4. **Import merge:** Use existing `addTab()` helper from storage.ts which already handles duplicate detection.
5. **Chrome APIs:** Use `chrome.downloads.download()` for export file save. Use HTML `<input type="file">` for import file selection.
6. **Accessibility:** Maintain aria-labels on the icon. Ensure options page is keyboard navigable.
7. **No changes to grid:** The TabGrid, TabCard, and related components remain exactly as implemented in Phase 3.
