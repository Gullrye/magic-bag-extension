# Phase 3 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | Tailwind CSS (manual) |
| Preset | not applicable |
| Component library | none (manual React components) |
| Icon library | favicon from tab data + inline SVG icons |

**Note:** Phase 3 inherits the actual warm chessboard surface already implemented in Phase 2, not the earlier neutral gray draft. The bag remains a Shadow DOM-contained panel with a warm amber board, white cards, blue interaction accent, and red destructive actions.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Fine icon spacing |
| sm | 8px | Compact control gaps |
| md | 16px | Default gaps between controls |
| lg | 24px | Panel padding |
| xl | 32px | Section separation |
| 2xl | 48px | Major empty-state padding |

**Inherited layout constraints:**
- Panel padding: 24px
- Card size: 160x100px
- Card gap: 16px
- Search/control row gap: 12px
- Search field height: 44px
- Action button minimum hit target: 44x44px

Exceptions:
- Card internal padding remains 12px to preserve current card proportions

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #FFFBEB (amber-50) | Bag panel surface |
| Secondary (30%) | #B45309 (amber-700) | Panel border, board texture emphasis |
| Accent (10%) | #3B82F6 (blue-500) | Focus ring, hover feedback, active drag emphasis |
| Destructive | #DC2626 (red-600) | Delete button, clear-all confirm action |
| Card Surface | rgba(255,255,255,0.98) | Tab cards, search field, dialog surface |
| Body Text | #374151 (gray-700) | Default text in cards and controls |

**Board texture:**
- Keep the existing subtle chessboard grid lines in the bag background
- Lines remain low contrast and decorative, never stronger than content

**Accessibility rules:**
- Interactive controls must show a visible focus state
- Icon-only controls require `aria-label`
- Destructive buttons must not rely on color alone; pair color with explicit copy/icon

## Component Inventory

### Grid Container (Phase 3 update)

**Component:** `TabGrid.tsx`

**Visual specification:**
- Keep current fixed amber board panel, rounded corners, and open animation
- Add a sticky top management row inside the panel
- Management row contains:
  - left: search field
  - right: clear-all button
- Results area remains below the control row and scrolls when needed

**States:**
1. Closed: not rendered
2. Open, query empty: show all saved tabs in persisted order
3. Open, query active with matches: show filtered cards
4. Open, query active with no matches: show search-empty state
5. Open, no saved tabs: show existing empty bag state

**Interaction behavior:**
- Opening the bag does not auto-focus the search field
- Opening the bag resets the query to empty
- Clear-all button is hidden or disabled when there are no saved tabs
- Clear-all acts on all saved tabs, not only filtered results

---

### Search Field

**Component:** inline in `TabGrid.tsx` or thin `SearchBar.tsx` if extraction improves readability

**Visual specification:**
- Height: 44px
- Width: fills available header width
- Background: white with high opacity
- Border: subtle amber/neutral border in resting state
- Radius: 10px
- Left search icon
- Body text size: 14px
- Placeholder uses muted text

**States:**
1. Idle: neutral border
2. Hover: slightly darker border
3. Focus: visible accent ring and stronger border
4. Filled: same layout, no auto-clear control required for Phase 3

**Interaction behavior:**
- Case-insensitive substring matching
- Matches title or URL
- No debounce in the first implementation

---

### Tab Card (Phase 3 update)

**Component:** `TabCard.tsx`

**Visual specification:**
- Keep 160x100px footprint, white surface, border, and hover shadow from Phase 2
- Add an explicit delete affordance in the card corner
- Delete affordance may be visually compact, but must provide a 44x44px hit target

**States:**
1. Idle: same as Phase 2
2. Hover: existing blue hover treatment
3. Delete hover/focus: destructive color treatment on the delete affordance
4. Dragging: card reduces accidental click ambiguity via drag activation threshold and visual drag state

**Interaction behavior:**
- Card body click still opens the tab
- Delete button never opens the tab
- Drag-to-reorder uses whole-card drag with activation distance so click remains easy

---

### Search Empty State

**Component:** inline in `TabGrid.tsx` or tiny dedicated component if needed

**Visual specification:**
- Centered within results region
- Same warm visual language as the bag
- Title: 16px semibold
- Body: 14px regular, muted
- Optional display of the current query in muted inline text

**Purpose:**
- Distinct from the true empty bag state
- Communicates "data exists, current filter removed it from view"

---

### Confirm Dialog

**Component:** `ConfirmDialog.tsx`

**Visual specification:**
- Fixed overlay inside the Shadow DOM
- Backdrop: black at ~40% opacity
- Dialog surface: white, rounded-xl, elevated shadow
- Width: max 360px, full width on small screens with safe padding
- Title and body stacked with 16px spacing
- Footer actions aligned right

**Buttons:**
- Cancel: neutral secondary button
- Confirm: destructive solid button
- Both buttons have minimum 44px height

**Interaction behavior:**
- Opened only for "clear all"
- Escape closes dialog
- Backdrop click may close dialog if implemented consistently with accessibility
- Initial focus should land on the safe action (`取消`) if practical

## Interaction Patterns

### Search and Reset
- User opens the bag and sees the full list
- User types in the search field to narrow results by title or URL
- Closing and reopening the bag resets the field and restores the full list

### Individual Delete
- User hovers or focuses a card and sees a clear delete affordance
- Clicking delete removes only that saved tab
- Delete action does not trigger tab open

### Clear All
- User clicks `清空`
- Confirmation dialog appears before destructive action
- Confirm removes all saved tabs and returns the panel to the true empty state

### Reorder
- User drags a card with a small movement threshold before drag starts
- Reordered cards persist after close/reopen and reload
- Reorder affordance must not make normal click-to-open unreliable
