## Interaction Patterns

### Card Hover Flow
1. User moves mouse over TabCard
2. Card scales to 1.02x, border changes to blue-500, shadow enhances
3. Delete button fades in over 150ms (opacity 0 -> 1)
4. User can click delete button to remove tab
5. User moves mouse away from card
6. Delete button fades out over 150ms (opacity 1 -> 0)
7. Card returns to default state (scale 1, gray border)

### Search with Sticky Header
1. User types in search input
2. Cards filter in real-time
3. User scrolls through filtered results
4. Search input and clear button remain visible at top
5. User can modify search without scrolling back up

### Delete Action
1. User hovers over card to reveal delete button
2. User clicks delete button
3. Card is removed from grid immediately
4. Storage is updated via removeTab helper
5. Remaining cards reflow to fill gap

## Animation Specifications

### Existing (unchanged)

| Animation | Duration | Easing | Properties |
|-----------|----------|--------|------------|
| gridOpen | 300ms | cubic-bezier(0.4, 0, 0.2, 1) | scale (0.8 -> 1), opacity (0 -> 1) |
| Card hover scale | 150ms | ease-out | scale (1 -> 1.02) |
| Card hover border | 150ms | ease-out | border-color |
| Card hover shadow | 150ms | ease-out | box-shadow |

### New

| Animation | Duration | Easing | Properties |
|-----------|----------|--------|------------|
| Delete button fade-in | 150ms | ease-out | opacity (0 -> 1), visibility (hidden -> visible) |
| Delete button fade-out | 150ms | ease-out | opacity (1 -> 0), visibility (visible -> hidden) |

**CSS implementation:**
```css
/* Delete button transitions */
.tab-card-delete-btn {
  opacity: 0;
  visibility: hidden;
  transition: opacity 150ms ease-out, visibility 150ms ease-out;
}

.tab-card:hover .tab-card-delete-btn {
  opacity: 1;
  visibility: visible;
}
```

**React implementation alternative:**
- Use `group` class on card wrapper and `group-hover:opacity-100 group-hover:visible` on delete button
- Or use React state with `onMouseEnter`/`onMouseLeave` handlers
