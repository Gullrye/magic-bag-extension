**Animation (from D-07):**

## Interaction Patterns

### Context Menu Collection

**Flow:**
1. User right-clicks anywhere on page
2. Chrome context menu appears with "将标签页收入法宝袋" option
3. User selects the option
4. Service worker captures current tab URL, title, favicon
5. Service worker checks for duplicate URL
6. If duplicate: Show warning toast, do not add
7. If new: Add to storage (newest-first order), close original tab, show success toast

**From CONTEXT.md decisions:**
- D-01: Context menu text
- D-02: No confirmation dialog
- D-03: Toast notification
- D-08: Storage schema { url, title, favicon, timestamp }
- D-09: Prevent duplicates
- D-10: Newest-first order

---

### Grid Toggle

**Flow:**
1. User clicks floating icon
2. Grid opens with scale+fade animation from icon position (300ms)
3. Grid displays all saved tabs in chessboard layout
4. User clicks outside grid OR presses Escape
5. Grid closes with scale+fade animation (200ms)
6. Floating icon returns to idle state

**From CONTEXT.md decisions:**
- D-04: Grid fills viewport then scrolls
- D-07: Animation scale+fade (300ms)
- GRID-05: Click outside to close

---

### Tab Restoration

**Flow:**
1. Grid is open, displaying tabs
2. User clicks a tab card
3. New browser tab opens with the saved URL
4. Grid remains open (user can open multiple tabs)
5. Optional: Grid could close after opening (Claude's discretion — recommend keeping open for rapid tab opening)

**From REQUIREMENTS.md:**
- GRID-04: Clicking tab opens URL in new tab

## Data Structure

**Tab schema (from D-08):**
```typescript
interface SavedTab {
  url: string;           // Full URL of the saved tab
  title: string;         // Page title (may be empty)
  favicon?: string;      // Favicon URL (optional)
  timestamp: number;     // Unix timestamp when collected
}
```

**Storage structure:**
```typescript
interface MagicBagStorage {
  tabs: SavedTab[];      // Array of saved tabs, newest-first
}
```

**From D-10:** Tab order is newest-first (most recently collected at index 0).

## Performance Considerations

**From CONTEXT.md discretion area:**

**Maximum storage limits:**
- chrome.storage.local has ~10MB default limit
- With `unlimitedStorage` permission (INFR-04), limit is ~100MB
- Each tab ~500 bytes (URL 100 + title 200 + favicon 200)
- Estimated capacity: ~20,000 tabs with unlimitedStorage
- Phase 2 does NOT implement pagination — render all tabs

**Rendering optimization:**
- Use React.memo() for TabCard to prevent unnecessary re-renders
- Use virtual scrolling only if tab count exceeds 100 (deferred to Phase 3 if needed)
- Lazy load favicon images (use `loading="lazy"` attribute)

## Accessibility

**Keyboard navigation:**
- Escape key closes grid
- Tab key focuses tab cards in order (left-to-right, top-to-bottom)
- Enter/Space on focused card opens URL
- Focus trap: Tab cycles within grid when open

**Screen reader support:**
- Grid container: `role="dialog"`, `aria-label="法宝袋"`
- Tab card: `role="button"`, `aria-label="{title}, {url}"`
- Toast: `role="status"`, `aria-live="polite"`

**Visual accessibility:**
- Minimum touch target: 44x44px (tab card meets this)
- Color contrast: All text meets WCAG AA (4.5:1 for normal text)
- Focus indicators: Blue outline (#3B82F6, 2px) on focused elements

## Implementation Notes

1. **Storage integration:** Extend existing `utils/storage.ts` with new `defineStorage` for tabs array
2. **Context menu:** Register in `entrypoints/background.ts` using `chrome.contextMenus.create`
3. **Grid positioning:** Calculate transform-origin based on floating icon position (from storage)
4. **Favicon fetching:** Use `chrome.tabs.get()` to capture favicon URL when collecting tab
5. **Duplicate detection:** Check `tabs.some(tab => tab.url === newUrl)` before adding
6. **Newest-first ordering:** `tabs.unshift(newTab)` to add at beginning of array
7. **Shadow DOM:** Grid must render within same Shadow Root as floating icon (from Phase 1)
8. **Z-index stacking:** Grid (2147483647) > Toast (2147483647) > Icon (2147483647) — use stacking context within Shadow DOM
